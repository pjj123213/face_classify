# 桌面端反馈闭环说明

## 1. 功能目的

桌面端新增 `预测错误` 按钮，供测试人员在模型预测错误时提交正确标签。

该功能不是直接在单次点击后立即强行修改模型，而是先把错误样本结构化保存下来，再通过反馈微调脚本进行小步训练。这样更稳，能避免单张误标样本对模型造成过大影响。

## 2. 桌面端使用流程

1. 选择图片；
2. 点击 `开始预测`；
3. 如果预测错误，点击 `预测错误`；
4. 在弹窗中选择正确标签；
5. 可选填写备注；
6. 点击 `保存反馈`。

保存后会生成：

```text
feedback/images/<正确标签>/feedback_*.jpg
feedback/feedback_log.csv
```

其中 `feedback_log.csv` 会记录：

- 时间；
- 原图路径；
- 保存后的反馈图片路径；
- 模型预测标签；
- 预测置信度；
- 测试人员给出的正确标签；
- 7 类概率分布；
- 备注。

## 3. 反馈微调

当反馈样本累积到一定数量后，可以运行：

```bash
python3 code/train_feedback.py
```

该脚本会：

- 加载当前主模型 `model/best_resnet18_faces.pth`；
- 读取 `feedback/images/` 中的反馈样本；
- 将原训练集和反馈样本一起训练；
- 只微调 `layer4 + fc`；
- 使用很小学习率 `5e-6`；
- 输出反馈版模型：

```text
model/best_resnet18_faces_feedback.pth
model/feedback_training_metrics.json
```

如果确认反馈模型指标更好，可以运行：

```bash
python3 code/train_feedback.py --promote
```

这会把反馈微调后的模型提升为桌面端默认使用的主模型。

## 4. 建议

不建议只用 1-2 张反馈样本就频繁覆盖主模型。更稳妥的做法是累计每类至少若干张反馈样本后，再运行反馈微调。

建议优先收集：

- `Latino_Hispanic` 被误判的样本；
- `Indian` 与 `Middle Eastern` 混淆样本；
- `East Asian` 与 `Southeast Asian` 混淆样本；
- 置信度较高但预测错误的样本。
